## Reproducible results

- `cd apps/api && .venv/bin/ruff format --check src tests alembic && .venv/bin/ruff check src tests alembic` — passed.
- `cd apps/api && .venv/bin/mypy src tests` — strict mode passed for 60 source files.
- `cd apps/api && .venv/bin/pytest -q` — **42 passed**.
- `cd apps/web && npm run format && npm run lint && npm run typecheck` — passed.
- `cd apps/web && npm test` — **12 passed in 7 files**.
- `cd apps/web && npm run build` — passed; 25 routes generated.
- `PLAYWRIGHT_BASE_URL=http://localhost:3001 npx playwright test --workers=1` — **8 passed**,
  desktop and mobile Chromium. Serial execution avoids development-server cold-compilation
  contention; the production build passed separately.
- `docker compose ps` — PostgreSQL, API, and web healthy on ports 3001 and 8001.
- `docker compose exec -T api alembic current` — `20260718_0007 (head)`.
- Fresh disposable PostgreSQL migration — all seven revisions applied to head, then the database was dropped.
- `docker compose exec -T api civicsignal-seed` — passed; six explicitly fictional services are public locally.
- `docker compose exec -T api civicsignal resources detect-stale --dry-run` — passed; six examined, no work created.
- Source/import tests — dry run created no records; committed test import created a candidate only; an authenticated human acceptance created a governed draft with no public service.
- Official 2026 HRSA staging — 18,953 active rows transformed into 38 files of at most 500 rows and 900 KB; all 18,953 candidates passed `DraftContent` validation; zero database or public records were created.
- `pip-audit --strict --format=json apps/api` — 34 dependencies, **0 vulnerabilities**.
- `npm --prefix apps/web audit --omit=dev --audit-level=high` — **0 vulnerabilities** at every severity.
- Secret-pattern and tracked-file checks — no credential pattern, tracked `.env`, SQL dump, or database dump found. Example environment templates are intentionally tracked.
- Docker Scout — **blocked** because the local Docker client requires account authentication.
- Backup/restore — an 80 KB custom-format backup restored to an isolated database at migration head. Counts: six services, one correction, one administrator, two sessions, four audit events, and zero approved sources/imports/candidates, matching the source database.
- Browser trust-page smoke — Coverage, Data Sources, Verification, About, Privacy, Accessibility, Security, resources, and administrator sign-in rendered at their expected URLs. The first cold development navigation can exceed the browser helper timeout while Next.js compiles; the rendered page was then present.
