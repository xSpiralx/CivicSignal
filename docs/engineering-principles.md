# Engineering principles

1. Accuracy outranks feature breadth. Missing information is preferable to fabricated information.
2. Provenance and verification timestamps are part of the domain, not optional metadata.
3. Collect the minimum personal data necessary and document retention before collection begins.
4. Design semantic, keyboard-operable, responsive pages that remain useful on constrained connections.
5. Prefer direct, typed code and explicit failures over hidden behavior or premature frameworks.
6. Keep application startup, shutdown, configuration, migrations, and operational health observable.
7. Pin direct dependencies, automate checks, and review upgrades rather than treating automation as assurance.
8. Humans approve architecture, security controls, data sources, and releases.
9. Claims about security, compliance, availability, or resource accuracy require evidence and authorization.
10. Tests should protect important behavior and failure paths, not merely increase coverage counts.
