# AI-assisted development policy

AI-generated code, tests, documentation, and plans are allowed without stigma. Material assistance must be disclosed in the pull request template, including the tool and affected areas. The contributor remains the author and is responsible for understanding, testing, licensing, security, accessibility, and accuracy.

AI output is untrusted until reviewed. Generated tests must exercise real behavior; fabricated citations, sources, test results, reviews, or command claims are prohibited. New dependencies and copied/generated material require provenance and license review. Security-sensitive, privacy, authentication, verification-policy, source-ingestion, migration, and release changes require additional qualified human scrutiny.

Never send secrets, production data, private user data, personal crisis information, embargoed vulnerabilities, or unnecessarily proprietary material to a coding tool. Maintainers may reject contributions that cannot be adequately explained, reproduced, supported, or safely reviewed. All architecture, security, source approval, and release decisions remain human responsibilities.
