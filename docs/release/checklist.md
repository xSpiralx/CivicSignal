# Alpha release checklist

- [ ] Version and changelog agree; tag is not published automatically
- [ ] CI, migrations from empty DB, upgrade path, seed, OpenAPI, Docker build, and demo pass
- [ ] Dependency, secret, SAST, response-leak, and license reviews pass
- [ ] Human security, privacy, source-policy, and threat-model review complete
- [ ] Keyboard, screen reader, zoom, contrast, reduced motion, touch targets, and error states manually checked
- [ ] Backup/restore, health, TLS, CSP, alerts, rollback, and documented environment verified
- [ ] Demo contains only unmistakably fictional data and source links

Beta additionally requires secure administration, controlled ingestion, re-verification operations, production rate limiting, restore exercises, and broader accessibility/security testing. Stable additionally requires sustained beta operations, incident and support processes, compatibility policy, measured reliability, and independent review appropriate to deployment risk.
