# CivicSignal v1.0 release report

Date: 2026-07-18

## 1–9. Executive and deployment summary

1. **Executive summary:** CivicSignal is a locally validated portfolio release candidate with a
   governed nationwide U.S. data foundation. It is not a live public-service directory.
2. **Platforms evaluated:** Vercel, Cloudflare Workers, Koyeb, Render, Neon, Supabase, and GitHub
   Actions. Official links and the comparison are in
   [`../operations/free-hosting-decision.md`](../operations/free-hosting-decision.md).
3. **Selected architecture:** Vercel Hobby Next.js, Render Free FastAPI, Neon Free PostgreSQL, and
   GitHub Actions CI/maintenance.
4. **Reason:** native Next.js compatibility, existing API container compatibility, a non-expiring
   free PostgreSQL path, TLS, secrets, and no authorized paid resource.
5. **Free-tier limits:** Vercel Hobby is personal/non-commercial and subject to hard limits; Render
   sleeps after 15 idle minutes and has an ephemeral disk; Neon has finite compute/storage and short
   free recovery history; scheduled GitHub workflows can be delayed or disabled after inactivity.
6. **Local Docker:** Docker 29.5.3 and Compose 5.1.4; database, API, and web healthy.
7. **Local validation:** 42 backend tests, 12 frontend tests, 8 browser tests, strict types, lint,
   formatting, Next.js build, existing/fresh migrations, seed, stale detection, dependency audits,
   backup, and restore passed. Full evidence is in
   [`../testing/v1.0-validation.md`](../testing/v1.0-validation.md).
8. **Deployment changes:** the free Render API Blueprint, Vercel config, shared-proxy secret,
   authenticated maintenance endpoint/workflow, production secret checks, trust pages, sitemap, and
   robots policy are prepared.
9. **Public deployment:** not performed; provider accounts are not authenticated in this checkout.

## 21–30. Governance and security controls

21. **Candidate categories:** public-health navigation, senior, veterans, food, employment, utility,
    and housing support.
22. **Human verification:** a named reviewer must confirm rights, official source, date, name,
    neutral description, geography, contact, hours, eligibility, cost, outcome, and next review date.
23. **High-risk safeguards:** authoritative sources, additional human review, visible source/date and
    emergency limits, no advice/ranking/case narrative, and immediate archival on strong closure
    evidence.
24. **Ingestion:** approved source → fetch/upload → parse → normalize → validate → deduplicate →
    preview → candidate → human-created governed draft → verification → separate publication.
25. **SSRF:** central HTTPS-only client, per-source hostname allowlist, public-IP DNS checks,
    connected-peer check, redirect revalidation/limit, timeouts, response/type limits, rate spacing,
    identified user agent, and safe logs.
26. **Imports:** 1 MB, 1,000 rows, UTF-8, MIME/extension agreement, safe filename, bounded columns,
    formula rejection, duplicate JSON key rejection, depth/scalar limits, row errors, content hash,
    retry protection, dry run, and batch transaction.
27. **Duplicates:** exact, likely, possible, new, or conflict. Only a new candidate can take the
    standard acceptance path; ambiguous records are never silently merged.
28. **Accessibility:** automated desktop/mobile tests found no serious or critical axe defects on
    scanned core routes. Manual keyboard, VoiceOver, and 200% zoom sign-off remains blocked.
29. **Security result:** package audits report zero known vulnerabilities; no secret pattern or
    tracked secret was found. Docker Scout is blocked on Docker account authentication.
30. **Browser result:** 8 expected, 0 unexpected, 0 skipped. The trust pages also rendered locally.

## 41–50. Release decision and handoff

41. **Remaining defects:** complete correction-to-publication browser fixture and full mobile admin
    pass are still missing; these are documented blockers, not concealed failures.
42. **Stable eligibility:** failed. Stable v1.0 requires live HTTPS, remote security/operations tests,
    verified real records, manual accessibility sign-off, and green final CI.
43. **Exact blockers:** GitHub, Vercel, Render, Neon, and Docker account authorization; named human
    source/record/high-risk/accessibility review; production administrator bootstrap; remote
    maintenance, monitoring, backup, cookie/CSRF/host/HTTPS tests; final CI.

    not create `v1.0.0`.
45. **Suggested title:** `CivicSignal v1.0 RC1 — Governed Nationwide Directory Foundation`.
46. **Suggested notes:** “Adds governed source approval, draft-only bounded imports, SSRF and parser
    safety, duplicate classification, public trust pages, and a no-cost deployment path. This RC uses
    fictional data; no real record is published until named human verification.”
47. **Local startup:** `cd /Users/robb/Documents/GitHub/CivicSignal && docker compose up --build -d`.
    Do not overwrite the existing `.env`. Open <http://localhost:3001>.
48. **Repository:** working tree intentionally contains this uncommitted milestone. No tag, push, or
    release was created and no credential was added.
49. **Portfolio recommendation:** show the fictional local demo and governance workflow now. Do not
    market it as live, comprehensive, production-ready, or populated with nationwide verified data.
50. **Maintenance:** check health/logs and free-tier usage weekly; run stale detection daily and
    backup monthly and before migrations; test restore quarterly; re-review source rights and every
    record by its due date; archive strong closure evidence immediately.

CivicSignal is deployment-ready, but external authorization is required for: GitHub, Vercel, Render, Neon, and Docker account access; a production administrator password; and named human source, resource, high-risk, and accessibility verification.
