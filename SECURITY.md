# Security policy

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability or include real personal crisis information in a report. Until a private security contact is configured, contact the repository owner privately through their established GitHub contact channel and provide only the minimum information needed to reproduce the issue.

Resource inaccuracies are normally handled through the resource-correction template unless disclosure could create immediate safety or abuse risk. Do not test systems you do not own or access other people's data.

The project will acknowledge reports when maintainer capacity permits, assess impact, and coordinate disclosure where appropriate. No response-time or remediation-time guarantee is currently offered.

## Supported versions

CivicSignal is pre-release software. Only the current default branch is considered for security fixes. It is not yet presented as a production service and is not claimed to be secure or compliant.
