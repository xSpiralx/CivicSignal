# Governance

Maintainers make decisions through documented review, favoring user safety, source integrity, accessibility, privacy, and maintainability. Routine changes require one maintainer approval; security-sensitive, architecture, data-source, verification-policy, and release changes require an additional qualified human review when the project has enough maintainers. Until then, such changes remain blocked rather than self-approved.

Maintainers may reject fabricated, harmful, unverifiable, copied, or inadequately explained resource submissions. Corrections and removal requests receive evidence-based review without guaranteeing a response time. Material decisions should be captured in issues or ADRs. Conflicts of interest must be disclosed, and the conflicted maintainer should recuse where practical.

Commit access is earned through sustained, constructive contribution and may be removed for inactivity or policy violations. This document can change through pull request and explicit maintainer approval.
